
# 検品と予定と総括 - README

## 想定ワークフロー

概ね、以下の順序で進行する

1. アニメ放映次クールの開始直前くらいに、次クールの `視聴予定.md` を作成
2. 1 話検品を行い `検品と総括.md` に 1 話寸評を記入
3. 検品結果を元に `視聴予定.md` を調整
4. 1 クール進行中でも必要に応じて視聴予定を都度調整
5. 1 クール全て見終わったら `検品と総括.md` に総括を記入

ただし、

- 放映開始・終了タイミングがバラバラなので上記の順序を同期的に進めるのは無理
- よって、以下のスタイルを取る
    - 各ステップはオーバーラップする
    - 埋められるところから埋めていくし、埋められないところは適当なプレースホルダにしておく
- e.g.
    - 前クールの総括が完了する前に、次クールの候補作成を開始する
    - 1 話検品が終わる前に視聴予定のドラフト作成を開始する

## ディレクトリ構造

- `検品と予定と総括/<year>/<season>`
- `<YEAR>`
    - 半角数字で西暦 4 ケタ
- `<SEASON>`
    - `C1`: 冬,  1 ~  3 月クール
    - `C2`: 春,  4 ~  6 月クール
    - `C3`: 夏,  7 ~  9 月クール
    - `C4`: 秋, 10 ~ 12 月クール
- e.g. `検品と予定と総括/2026/S3`

## `<season>` ディレクトリ内

### `検品リスト.md`

- リアタイ用の情報なので、このファイルが存在するのは 2026 年以降のみ
- 村の会議で決定して、そのクールのアニメの 1 話検品リスト
- 見出しでカテゴライズされる
    ```markdown
    # <YEAR> <SEASON>アニメ - 検品リスト

    ## <カテゴリ 1>

    ...

    ## <カテゴリ 2>

    ...

    ## ...

    ...

    ```
- 各カテゴリ内でアニメが箇条書きでリストアップされる
- アニメリストのフォーマットは以下の通り
    ```markdown
    - <月～日>曜, HH:MM, [<配信サイト名>](<配信URL>), [公式](<アニメ公式URL>), **<アニメタイトル>**, <補足事項>
    ```
- e.g. 肋骨なら
    ```
    - 火曜 26:30, [dアニ](https://animestore.docomo.ne.jp/animestore/ci_pc?workId=28727), [公式](https://rokkotsu-anime.com/), **『女神「異世界転生何になりたいですか」俺「勇者の肋骨で」』**, 神アニメ
    ```
- `<月～日>曜, HH:MM`
    - 時刻は `[05:00, 29:00)` で表記する
    - e.g. `火曜, 02:00` では無く `月曜, 26:00` と書く
- `<補足事項>`
    - 書いても良いし、書かなくても良い

### `視聴予定.md`

- リアタイ用の情報なので、このファイルが存在するのは 2026 年以降のみ
- 村の会議で決定した、そのクールのアニメの視聴予定
    - 最新の情報だけを書く（経緯とかはできるだけ書かない）
    - どの枠（みんなで、競技スクショ部、各自自習、…）で見るかでカテゴリ分けする
    - 何曜日の何時ごろに集まるかを書く方が望ましい
    - 見ない予定のアニメは書かない（余計な情報）
- 見出しによるカテゴライズは `検品リスト.md` と同じ
    - ただしタイトルは `# <YEAR> <SEASON>アニメ - 視聴予定` とする
- アニメリストのフォーマットは `検品リスト.md` と同じ

### `検品と総括.md`

- 「アニメ 1 話視聴時点での検品結果」と「1 クール見終わった後の総括」をまとめる
- 見出しで構造化される
    ```
    # <YEAR> <SEASON>アニメ - 検品と総括

    ## <アニメ 1>

    ### 1 話寸評

    ...

    ### 総括

    ...

    ## <アニメ 2>

    ### 1 話寸評

    ...

    ### 総括

    ...

    ```
- 「1 話寸評」「総括」の内容は自由記述
    - 箇条書きスタイルを想定

## 1 話検品のレビュー観点

### 基本的な考え方

- 「えぃにめ鑑賞村で」「みんなで」見る理由の有無が大事
- 1 話時点で判断しうることだけを観点にする

## 見る理由になる

- スクショ競技性は理由になる
    - 変な動きをする女がいるかどうか
- 「変さ」「トンチキ味」が有るアニメはプラス要因
    - 演出が妙な感じとか
    - 標準からズレてそうな臭い
    - コンテのうまさ
- 「普通に面白い罪」は理由になる
    - 「普通に面白いアニメ」はみんな黙ってしまう
    - こういう系は１話で概ねわかるので、高精度で除外出来る
- キャラ萌えは理由になる
    - Sana Logic 判定器を使えば OK
    - トンチキ女が大事

## 見る理由にならない

- エロがあるだけでは見る理由にならない
    - 実際、よわよわ先生は候補にも入らなかったし、振り返ってもそれは正解だった
- ワイワイ見るのに良さは理由にならない
    - 言い換えれば、話のシンプルさ
    - これは消極的理由
- 無味かつ引きが弱いのはマイナス要因だが理由にはならない
    - １話でも判定出来そう？
    - でも、お気楽領主は１話の引き良かったので、確実ではない
    - 判断基準の１つとしては使えそう
    - しかし、理由と出来るほど強い要素ではない

## そもそも観点に入れない

- ストレス味は、検品するまでもなくわかる事なので、観点には入れない

## ２話以降まで判断を持ち越したい場合

- 過去、１話トップスピードだったアニメにも、１話時点で光るものはあった
- １話判定を諦めて、自習した結果に基づいて敗者復活戦をする方が良い
